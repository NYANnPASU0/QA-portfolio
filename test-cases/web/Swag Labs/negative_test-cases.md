## Тест-кейсы для проверки негативных сценариев

### 
| ID тест-кейса | SL01_NEG <br>|
|:--------------|:---------|
|  Заголовок  | Оформление заказа: некорректный почтовый индекс(Zip/Postal Code). <br>|
| Предусловие | 1. Пользователь авторизован под любой допустимой учетной записью.<br> 2.В корзине лежит минимум один товар. <br> 3. Открыта страница ввода данных для оформления заказа(Checkout: Your Information). <br> |
| Шаги | 1. Ввести "Test" в поле First Name. <br> 2. Ввести "Test" в поле Last Name. <br> 3. В поле Zip/Postal Code ввести символ "1".<br> 4. Нажать на кнопку Continue. <br> |
| Ожидаемый результат | 1. Введенные данные в полях First Name и Last Name корректно отображаются. <br> 2. При вводе однозначного почтового индекса в поле Zip/Postal Code нельзя перейти к следующему этапу оформления заказа(Checkout: Overview). Отображается сообщение об ошибке о некорректном формате почтового индекса. <br> |
| Фактический результат | 1. Введенные значения заполнены корректно. [Скриншот: фио](images/SL01_NEG_lastname.png)<br> 2. В поле Zip/Postal Code введено значение "1". [Скриншот: почтовый индекс](images/SL01_NEG_postal_code.png)<br> 3. При нажатии на кнопку "Continue" выполнен переход на страницу Checkout: Overview. Сообщение об ошибке отсутствует.[Cкриншот: баг - переход на Overview без ошибки](images/SL01_NEG_fail.png) <br> |
| Постусловие |  Корзина очищена. Выполнен выход из системы(Logout). <br> |
| Статус | Failed |

| ID тест-кейса | SL02_NEG <br>|
|:--------------|:---------|
|  Заголовок  | Отказ в оформлении заказа при незаполненных обязательных полях. <br>|
| Предусловие | 1. Пользователь авторизовался на сайте https://www.saucedemo.com/. <br> 2. В корзине лежит несколько выбранных товаров. <br> 3. Открыта страница "Checkout: Your Information". <br> |
| Шаги | 1. Поле ввода "First Name" оставить пустым. <br> 2. Поле ввода "Last Name" оставить пустым. <br> 3.Поле ввода "Zip/Postal Code" оставить пустым. <br> 4. Нажать кнопку "Continue". <br> |
| Ожидаемый результат | 1. При нажатии на "Continue" редирект на следующую страницу не происходит. <br> 2. Все поля подчеркнуты красной полоской, справа отобразилась иконка в виде крестика. <br> 3. На экране появилось сообщение об ошибке: "Error: First Name is required".  <br> |
| Фактический результат | 1. При нажатии на "Continue" редирект не произошел. <br> 2. Поля ввода подчеркнуты красным с иконкой ошибки <br> 3. Видна ошибка: "Error: First Name is required". <br> [Скриншот: сообщение об ошибке](images/SL04_POS_checkout.png) <br> |
| Постусловие | Пользователь остаётся на странице ввода данных <br> |
| Статус | Passed |

Пароль в поле Password отображается в маскированном виде. [Скриншот: поле Password](images/SL03_NEG_password.png) <br> 3. При нажатии на кнопку "Login" редирект на страницу каталога товаров не произошел. На экране появилась ошибка "Epic sadface: Username and password do not match any user in this service". Поля ввода подчеркнуты красным, рядом с ними иконки с крестиком. [Скриншот: отказ в авторизации](images/SL03_NEG_not_correct.png)<br> |
