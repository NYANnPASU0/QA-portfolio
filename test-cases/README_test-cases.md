## Test Cases

В данной папке находятся тест-кейсы, написанные для разных сайтов, в том числе специально созданных для тестировщиков.

### Содержание 
| Папка | Описание |
|:------|:---------|
| web | Веб-приложения |
| mobile | Мобильные приложения |
| api | API-тестирование |
| database | Базы-данных |

### Стандарт оформления
Каждый тест-кейс следует структуре:

| Поле | Описание |
|:-----|:---------|
| ID тест-кейса | Уникальный идентификатор, который может быть числовым, буквенным, буквенно-численным |
| Заголовок | Наименование тест-кейса |
| Приоритет | High / Medium / Low |
| Тип | Smoke / Regression / Positive / Negative |
| Предусловия | Состояние системы и данные, необходимые до запуска |
| Шаги | Описание последовательных действий, которые необходимо выполнить для проведения теста |
| Ожидаемый результат | То, как система должна реагировать на шаги теста |
| Фактический результат | Фактический результат того, как отработала программа |
| Постусловия | Возможные действия по приведению системы в первоначальный вид |

### Форматы
- `.md` - для быстрого просмотра прямо на GitHub
- `.xlsx` - для импорта в TestRail / Allure TestOps
